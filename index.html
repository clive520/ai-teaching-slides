零、 AI 核心概念解析

建立最新的 AI 認知：生成式 AI vs AI Agent

👉 教學指引 (Step-by-Step)

1. 開場破冰：先問台下學生「大家平常怎麼用 ChatGPT？」，引導出目前多數人把 AI 當作單純的問答機器。

2. 核心切入：告訴學生今天這堂課不一樣，我們要教的是如何讓 AI 幫你「做事」，也就是進入 AI Agent 的領域。

1. 生成式 AI (Generative AI)

  • 定位：超級大腦或百科全書
  • 特性：一問一答。沒有手腳，需依賴人類指令推進。
  • 代表案例：ChatGPT, Google Gemini, Claude
  • 實用工具：Gemini「Gems」、NotebookLM

👉 教學指引 (Step-by-Step)

1. 展示 NotebookLM：點開事先準備好的 NotebookLM 網址。

2. 操作示範：在 NotebookLM 中上傳一份 PDF，並當場示範點擊「語音導覽 (Audio Overview)」，讓學生聽 AI 怎麼把無聊的講義變成兩人對話的 Podcast，引起學習興趣。

2. AI Agent (代理 / 智能體)

  • 定位：有手有腳的獨立助理
  • 特性：具備規劃任務、長期記憶、使用工具 (Tool Use)、與自主行動的能力。
  • 優勢：只要給定「大目標」，Agent 能自行拆解步驟、完成任務。

👉 教學指引 (Step-by-Step)

1. 比喻法：生成式 AI 是大腦，AI Agent 是「大腦 + 手 + 腳」。

2. 舉例說明：舉例「請幫我規劃日本五天旅遊並訂機票」。ChatGPT 只會給你計畫清單；但 Agent 會自己去查機票網、比對價格、並自動呼叫購票系統訂票。

3. 提示詞 (Prompt) 5 大要素

  1. 任務 (Task)：核心目標。
  2. 輸入資料 (Input Data)：背景資訊或素材。
  3. 輸出成品 (Output Format)：最終期望的格式。
  4. 限制條件 (Constraints)：範圍限制或語氣要求。
  5. 驗收標準 (Acceptance Criteria)：及格或完美的標準。

👉 教學指引 (Step-by-Step)

1. 發放學習單：請學生拿出一張紙或開啟筆記軟體。

2. 實作練習：請學生依照這 5 大要素，寫出一個「請 AI 幫忙規劃英文單字背誦計畫」的 Prompt。

3. 隨機抽點：請 2-3 位同學分享他們的 Prompt，並由老師指出哪裡可以加入更明確的「限制條件」或「驗收標準」。

4. 自動化任務的 5 大判準

不是每件事都值得做成 Skill！

  • 流程固定：步驟 SOP 化
  • 輸入明確：初始條件清楚
  • 成品明確：產出格式可預期
  • 重複發生：一再出現才有投資效益
  • 可由人類審核：產出能快速驗證對錯

👉 教學指引 (Step-by-Step)

1. 情境反問：「如果請 AI 每天幫你決定早餐吃什麼，適合做成自動化嗎？」(引導學生思考：這沒有固定 SOP，也不好驗證對錯，其實不適合。)

2. 重點強調：自動化的核心是「重複發生且 SOP 化」，不是為了用 AI 而用 AI。

5. 實用 AI Agent 案例

  • Google Antigravity：強大開發與系統 Agent。
  • ChatGPT Code Interpreter：進階資料分析，自動寫 Python。
  • Open Code：本機端開源 Agent。

一、 網頁與服務部署空間

成果展示與實作

👉 教學指引 (Step-by-Step)

1. 觀念釐清：告訴學生，網頁寫好只在自己的電腦裡，別人看不到。我們需要「部署」讓它上網。

2. 名詞解釋：用「租店面」來比喻網頁部署空間，網址就是「門牌號碼」。

1. 靜態網頁部署空間

  • GitHub Pages：經典免費，學習 Git 必備。
  • Vercel：前端最熱門，現代化 CI/CD。
  • Netlify：老牌無伺服器託管。
  • Cloudflare Pages：極速與超大流量。

👉 實作演練：GitHub Pages 詳細部署步驟

1. 請學生登入 GitHub。點擊右上角 `+` -> `New repository`。

2. 命名為 `my-first-ai-web`，設為 Public。

3. 點擊 `Upload files`，上傳事前寫好的 HTML 檔案，然後按下 Commit。

4. 點擊專案上方的 `Settings` -> 左側找到 `Pages`。

5. 在 Branch 下拉選單選擇 `main`，點擊 `Save`。

6. (提醒學生) 等待大約兩分鐘，頁面上方就會出現專屬網址囉！

2. 結合資料庫的動態網頁

  • GAS + Google Sheets：最親民，試算表當後端。
  • Vercel + Supabase：最主流現代全端。
  • Firebase：強大 NoSQL 與即時資料庫。
  • Render / Zeabur：容器化與 PostgreSQL 部署。

👉 教學指引 (Step-by-Step)

1. 痛點說明：靜態網頁無法存資料 (例如留言板)。如果要存資料，入門非常推薦用 GAS + Google 試算表。

2. 展示範例：打開一個已經寫好的 Google 表單 + 試算表，當場示範資料是如何即時存進免費的試算表中的。

3. CI/CD 自動化與限制

  • GitHub 自動化上傳，免 FTP。
  • 注意限制：冷啟動 (Cold Start)、流量上限。
  • 非永續性風險：請定期備份！

👉 實機展示 CI/CD (自動化部署)

1. 在本地端用 VS Code 打開剛剛上傳的 HTML 檔，當著學生的面修改一個大標題文字。

2. 打開 GitHub Desktop，點擊 `Commit` 跟 `Push origin`。

3. 馬上切換到 GitHub 網頁，讓學生看 Actions 正在自動編譯的齒輪轉動畫面。

4. 結論：向學生證明，有了 CI/CD，以後只要修改程式碼，不需要再手動開網頁上傳，網站就會自動更新！

二、 學生/一般 AI 教學範例

實作範例 (1)

  • 呈現文字雲：萃取長文關鍵字，利用 AI 寫 Python / JS 生成視覺圖。
  • 製作教學助手：使用 Custom GPTs 或 Dify (RAG) 打造專屬助教。
  • HTML 簡報：AI 生成 Markdown，直接無痛轉換為 Reveal.js。

實作範例 (2)

  • 打造行事曆 Agent：使用 No-Code 平台串接 Google Calendar，解析自然語言排程。
  • API 串接：
    • 免費：Gemini, NVIDIA NIM, Groq
    • 付費：OpenAI, Claude, OpenCode (GO)

安全提醒：API Key 絕不能直接寫在前端公開代碼中！

三、 專屬教師的備課範例

自動化備課流程

  • 教案製作：快速生成含動機、發展、評量的草案。
  • 建立題庫：試題 PDF 轉文字，結構化出題。
  • 會議紀錄：語音 STT 轉文字，清洗萃取決議事項。
  • 問卷調查：生成題目並透過 Tally.so 直接轉表單。

打造教師專屬「第二大腦」

  • 利用 Obsidian 建立本地端結構化知識庫。
  • 讓 AI (NotebookLM / Agent) 存取資料夾，根據個人脈絡回答。
  • 雙向反饋：AI 產出之新教材，再存回 Obsidian 擴充大腦。

四、 好用工具推薦

  • Typeless：AI 聽打與會議整理神器。
  • Gamma / Tome：輸入一句話，生成精美網頁簡報。
  • Napkin AI：文字瞬間轉為流程圖。
